- Semantic Versioning 2.0.0: https://semver.org/
  - Major versions indicate incompatible API changes, minor versions add backwards-compatible functionality, and patch versions make backwards-compatible fixes.
- Keep a Changelog: https://keepachangelog.com/en/1.1.0/
  - A changelog is a curated, chronological list of notable changes. Keep an `Unreleased` section for upcoming changes.
- npm package.json docs: https://docs.npmjs.com/files/package.json/
  - `package.json` is the standard place for package metadata such as version in npm projects.
- npm scripts docs: https://docs.npmjs.com/cli/using-npm/scripts/
  - npm scripts define project commands used for validation and release checks.
- GitHub Actions docs: https://docs.github.com/actions
  - GitHub Actions can automate build, test, and deployment workflows.
