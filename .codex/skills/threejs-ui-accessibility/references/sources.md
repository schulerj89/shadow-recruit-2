# Source Notes

Research date: 2026-06-19.

- W3C WCAG 2.2 Quick Reference: https://www.w3.org/WAI/WCAG22/quickref/
  - Use WCAG success criteria and techniques as the baseline for accessible web UI.
- W3C ARIA Authoring Practices Guide patterns: https://www.w3.org/WAI/ARIA/apg/patterns/
  - Provides accessible patterns for widgets such as dialogs, menus, buttons, tabs, radio groups, and more.
- W3C Modal Dialog Pattern: https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/
  - Modal dialogs keep focus within the dialog until closed.
- W3C Focus Visible understanding: https://www.w3.org/WAI/WCAG22/Understanding/focus-visible.html
  - Users need a visible indication of keyboard focus.
- W3C Focus Not Obscured: https://www.w3.org/WAI/WCAG22/Understanding/focus-not-obscured-minimum.html
  - Focused items should remain at least partially visible.
- MDN Gamepad API: https://developer.mozilla.org/en-US/docs/Web/API/Gamepad_API
  - Provides controller connection events and access to buttons and axes.

Re-check current accessibility guidance before changing UI patterns or input support.
